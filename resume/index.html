RAG 知识库问答系统 独立开发
2026.04 - 至今
项目概述
基于 RAG 技术实现基于文档内容的精准问答，支持在线演示（预置文档）与本地部署（自定义文档）双模式。
核心功能
支持 PDF/TXT/MD/DOCX 多格式文档上传；查询长度自适应的混合检索；两阶段排序优化检索结果相关性。
Vue3 Vue Router Node.js Express 向量相似度计算 LLM API集成 SSE流式传输
  • 多格式文档解析兼容性：PDF 和 DOCX 解析时遇到中文乱码、表格丢失等问题。通过组合使用 pdf-parse 和 mammoth.js，针对不同格式选择最优解析器。上传环节通过 iconv-lite 实现文件名编码自动检测，支持 UTF-8、GBK、GB2312 等多种编码转换，解决了中文文件名乱码问题。
  • 检索召回率低，答非所问：针对向量检索对专业术语召回差、停用词过滤过度问题，采用查询长度自适应混合检索（长查询：向量60%+关键词30%+人名10%；短查询：向量50%+关键词40%+人名10%），并优化停用词表。增加两阶段排序：先召回Top-30，再用关键词覆盖度Rerank（混合分数80%+覆盖度20%），显著提升相关性。
  • 大模型 API 调用不稳定：向量化和 LLM 推理依赖外部 API，网络波动导致请求失败。实现指数退避重试机制（最大重试 3 次，基础延迟 1 秒，最大延迟 10 秒），并对错误进行分类处理（网络错误和超时自动重试、认证错误和参数错误立即返回），提升系统稳定性。同时接入 express-rate-limit 做接口限流，防止恶意调用。
  • SSE 长连接断开：RAG 问答采用 SSE 流式返回答案，长时间无数据传输导致连接被代理服务器断开。增加心跳保活机制（每 15 秒发送一次心跳包），保持连接稳定。

AI 写作助手 独立开发
2026.03 - 至今
项目概述
解决长文写作大纲构思难、扩写耗时问题，实现从大纲生成到分节扩写的完整流程。
核心功能
输入主题自动生成结构化大纲；分节内容一键生成，支持润色/扩写/缩写；打字机效果展示，支持随时中断。
Vue3 Pinia Vite Tailwind CSS Node.js Express LLM API集成
  • 大模型响应延迟导致用户体验差：首次调用 LLM 生成大纲时响应较慢，用户等待焦虑。通过引入 SSE 流式传输，将生成过程分块返回，配合前端打字机动画逐字展示，用户可提前阅读内容。同时基于 AbortController 实现请求中断控制，用户可随时停止生成，避免长时间无意义的等待。
  • 长文本生成时前端渲染卡顿：扩写长内容时，频繁的 DOM 更新导致页面滚动和交互卡顿。采用分章节逐个生成策略，每次只渲染当前章节内容，配合 Vue 的 shallowRef 减少不必要的响应式追踪，保持页面交互流畅。
  • 大纲与正文内容结构不一致：用户修改大纲后重新生成正文，容易出现前后内容矛盾。采用大纲驱动的生成策略，每次生成都基于当前最新大纲结构，确保正文内容与大纲保持一致。
